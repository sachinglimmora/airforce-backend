IAF Training - API Documentation

Base URL: http://localhost:5000/api

Auth: All routes (except login) require a Bearer token in the Authorization header.

Authentication

POST /api/auth/login Public

Authenticate user and get JWT token.

Request Payload

{
  "email": "arjun.singh@iaf.gov.in",
  "password": "password"
}

Response (200 OK)

{
  "token": "eyJhbG...",
  "user": {
    "id": "1",
    "email": "arjun.singh@iaf.gov.in",
    "role": "trainee",
    "name": "Flight Lt. Arjun Singh",
    "rank": "Flight Lieutenant"
  }
}
GET /api/auth/me Bearer token required

Get current user's profile.

Response (200 OK)

{
  "id": "1",
  "email": "arjun.singh@iaf.gov.in",
  "name": "Flight Lt. Arjun Singh",
  "role": "trainee",
  "rank": "Flight Lieutenant"
}

Users

GET /api/users Admin / Instructor

List all users. Supports ?role=trainee & ?search=John

Response (200 OK)

[
  {
    "id": "...",
    "name": "...",
    "email": "...",
    "role": "trainee"
  }
]
POST /api/users Admin only

Create a new user.

Request Payload

{
  "name": "New User",
  "email": "new.user@iaf.gov.in",
  "password": "password123",
  "role": "trainee",
  "rank": "Flying Officer",
  "base": "Delhi"
}

Response (201 Created)

{
  "id": "...",
  "name": "New User",
  "email": "new.user@iaf.gov.in",
  "role": "trainee"
}

Courses

GET /api/courses Any logged-in user

List all training courses. Query params: ?category=&status=&difficulty=

Response (200 OK)

[
  {
    "id": "1",
    "title": "Su-30MKI Engine Systems",
    "category": "Jet Engine Systems",
    "difficulty": "advanced",
    "moduleCount": 12,
    "completedModules": 8,
    "status": "in-progress"
  }
]
POST /api/courses Admin / Instructor

Create a new training course.

Request Payload

{
  "title": "New Navigation Training",
  "category": "Navigation",
  "description": "Learn advanced navigation concepts.",
  "difficulty": "intermediate"
}

Response (201 Created)

{
  "id": "...",
  "title": "New Navigation Training",
  "category": "Navigation",
  "status": "not-started"
}

Modules

GET /api/modules Any logged-in user

List modules. Query constraint: ?courseId=1

Response (200 OK)

[
  {
    "id": "m1",
    "courseId": "1",
    "title": "AL-31FP Engine Overview",
    "isCompleted": true,
    "procedures": [...],
    "diagrams": [...]
  }
]
POST /api/modules/:id/complete Any logged-in user

Mark a module as complete. This auto-calculates the relative course progress and updates the trainee's readiness scores.

Response (200 OK)

{
  "message": "Module marked as complete.",
  "module": {
    "id": "m1",
    "isCompleted": true
  }
}

Simulations

GET /api/simulations Any logged-in user

List simulations. Query params: ?type=&difficulty=&status=

Response (200 OK)

[
  {
    "id": "sim1",
    "title": "Engine Fire Emergency",
    "type": "maintenance",
    "difficulty": "advanced",
    "status": "available"
  }
]
POST /api/simulations/:id/start Any logged-in user

Start a simulation. Sets status to `in-progress`.

Response (200 OK)

{
  "message": "Simulation started.",
  "simulation": {
     "id": "sim1",
     "status": "in-progress"
  }
}
POST /api/simulations/:id/complete Any logged-in user

Complete a simulation run. Updates trainee simulation hours and saves their score.

Request Payload

{
  "score": 95
}

Response (200 OK)

{
  "message": "Simulation completed.",
  "simulation": {
     "id": "sim1",
     "status": "completed"
  }
}

Instructor Analytics

GET /api/analytics Instructor / Admin

Returns aggregation of completion rates, trends, and simulations used across trainees.

Response (200 OK)

{
  "summary": {
    "totalTrainees": 15,
    "avgReadiness": 72,
    "totalSimHours": 320.5
  },
  "charts": {
     "trainingCompletion": [...],
     "readinessTrend": [...]
  }
}

Progress

GET /api/progress Any logged-in user

Get progress records. Trainee gets own progress; Instructor/Admin gets all.

Response (200 OK)

[
  {
    "traineeId": "1",
    "readinessScore": 78,
    "overallProgress": 58,
    "simulationHours": 42
  }
]
PATCH /api/progress/:traineeId Instructor / Admin

Update trainee progress manually.

Request Payload

{
  "readinessScore": 85,
  "simulationHours": 50
}

Digital Twin

GET /api/digital-twin Any logged-in user

Get list of all functional aircraft systems. Query params: ?category=&status=

Response (200 OK)

[
  {
    "id": "sys1",
    "name": "AL-31FP Engine",
    "category": "engine",
    "health": 94,
    "status": "operational"
  }
]
PATCH /api/digital-twin/:systemId/components/:componentId Instructor / Admin

Update a component's health status. Auto-recalculates the overall system health.

Request Payload

{
  "health": 60,
  "status": "maintenance"
}

Response (200 OK)

{
  "message": "Component updated. System health recalculated.",
  "system": {
    "id": "sys1",
    "health": 85
  }
}

AI Assistant

POST /api/ai-assistant/message Any logged-in user

Send a query to the AI Assistant and receive an expert response.

Request Payload

{
  "content": "What is the emergency procedure for an engine fire?"
}

Response (200 OK)

{
  "response": {
    "role": "assistant",
    "content": "In case of an engine fire indication on the Su-30MKI, follow these emergency procedures...",
    "timestamp": "2024-04-02T12:00:00Z"
  }
}

Alerts

GET /api/alerts Any logged-in user

Fetch system alerts. Query filter: ?type=critical&unread=true

Response (200 OK)

[
  {
    "id": "alert1",
    "type": "critical",
    "title": "System Maintenance",
    "isRead": false
  }
]
PATCH /api/alerts/:id/read Any logged-in user

Mark a specific alert as read.

PATCH /api/alerts/read-all Any logged-in user

Mark all open alerts as read.
